Product Requirements Document (PRD) & Business RulesProyecto: Plataforma de Monitoreo de Turbidez - Bahía de CartagenaFase Actual: MVP (Fase 1 - Carga Manual de Datos)Infraestructura: On-Premise (Datacenter Universitario)1. Visión GeneralEl sistema es una plataforma web Full-Stack (React + FastAPI) para visualizar, analizar y gestionar datos de turbidez del agua en la Bahía de Cartagena, Colombia. En su Fase 1 (MVP), el sistema ingerirá archivos JSON con datos de reflectancia ($Rrs\_665$) y un valor de turbidez ya precalculado mediante un algoritmo propio de Support Vector Regression (TT_pred). La aplicación proyectará estos datos en un mapa interactivo (Heatmap).2. Restricciones Críticas (Guardrails para Agentes)CERO NUBE PÚBLICA: No utilizar AWS, GCP ni Azure. Todo el despliegue es On-Premise usando Docker.ALMACENAMIENTO DE DATOS: La persistencia principal es PostgreSQL con extensiones PostGIS (espacial) y TimescaleDB (series temporales).CERO RASTERS EN FASE 1: No procesar archivos .tif ni .nc en esta fase. La ingesta es puramente vectorial (Puntos Lat/Lon) vía archivos JSON.TURBIDEZ PRECALCULADA (FASE 1): Para el algoritmo principal (SVR), la base de datos guarda el valor de turbidez final (TT_pred). El Backend (FastAPI) solo debe extraer este valor y enviarlo al Frontend, sin aplicar cálculos matemáticos. La arquitectura debe quedar lista para futuros algoritmos que sí requieran cálculo "on-the-fly" usando el $Rrs\_665$.3. Tipos de UsuariosPúblico General / Investigador: Accede al mapa, selecciona fechas, selecciona algoritmos (solo SVR disponible en Fase 1) y visualiza heatmaps y series temporales.Administrador: Tiene acceso a un panel protegido donde puede subir archivos JSON de nuevas fechas para alimentar la base de datos de 5 años.4. Volumetría de Datos (MVP)Puntos por día: ~700.Histórico a cargar: 5 años (aprox. 1,825 días).Total estimado de registros en BD: ~1.27 Millones de puntos. (Carga ligera, pero requiere partición por tiempo para consultas rápidas).
