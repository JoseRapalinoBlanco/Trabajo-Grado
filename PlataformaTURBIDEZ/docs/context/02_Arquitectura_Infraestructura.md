Arquitectura del Sistema (On-Premise)Este documento define la topología de contenedores y servicios. El sistema debe ser orquestado mediante docker-compose (preparado para Docker Swarm o K3s).1. Componentes del Sistema (Contenedores)1.1. Reverse Proxy (Traefik / Nginx)Rol: Puerta de enlace. Enruta peticiones /api/* al Backend y / al Frontend.Puertos: Expone 80 y 443 al exterior.1.2. Frontend (React.js + Vite)Rol: SPA (Single Page Application). Interfaz de usuario y renderizado de mapas.Librerías Clave: OpenLayers (para el mapa base y renderizado WebGL del Heatmap), Zustand o Redux Toolkit (estado), TailwindCSS (UI).1.3. Backend (Python + FastAPI)Rol: API RESTful asíncrona. Maneja ingesta de archivos, lógica de negocio y cálculos bio-ópticos vectorizados.Librerías Clave: FastAPI, SQLAlchemy o asyncpg, GeoPandas, NumPy, Pydantic (validación de JSON).1.4. Base de Datos (PostgreSQL + PostGIS + TimescaleDB)Rol: Almacenamiento transaccional, espacial y temporal.Imagen Docker: Usar la imagen oficial de Timescale que incluye PostGIS: timescale/timescaledb-ha:pg15-latest (o similar compatible).1.5. Object Storage local (MinIO)Rol: Preparación para Fase 2. En Fase 1 se usará para guardar un backup inmutable de los archivos JSON originales que suba el administrador, antes de ser procesados por la BD.2. Flujo de Red (Network Flow)Usuario -> Reverse Proxy -> Frontend (Carga UI).Frontend -> Reverse Proxy -> Backend (Petición de Heatmap GeoJSON).Backend -> PostgreSQL (Query SQL espacial/temporal).PostgreSQL -> Backend (Retorna Puntos con Rrs_665).Backend (Aplica Algoritmo en memoria) -> Frontend (GeoJSON con Turbidez).
