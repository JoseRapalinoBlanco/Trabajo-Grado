Especificaciones de la API (FastAPI)El agente backend debe implementar los siguientes endpoints principales. Todos deben tener prefijo /api/v1.1. Endpoints de Ingesta (Protegidos por Auth)POST /api/v1/admin/upload-dataDescripción: Recibe el archivo JSON con los datos espaciales, de reflectancia y turbidez precalculada.Payload: multipart/form-data (archivo JSON).Lógica:Guardar copia en MinIO (backup).Parsear JSON con Pydantic.Ejecutar Bulk Insert en TimescaleDB extrayendo Longitude, Latitude, Rrs_665 y TT_pred.Respuesta: {"status": "success", "records_inserted": 700, "date": "2024-01-14"}2. Endpoints de Consulta (Públicos)GET /api/v1/turbidity/heatmapDescripción: Retorna los puntos de una fecha específica con la turbidez para renderizar en el mapa.Query Params:date (YYYY-MM-DD): Fecha requerida.algorithm (string): Enum [SVR]. (Preparado para agregar más en el futuro).Lógica Interna (Crítica):Validar el algorithm.Si algorithm == 'SVR': Hacer un SELECT geom, tt_pred as turbidity_ntu FROM turbidity_data WHERE measurement_date = {date}.(Futuro: Si es otro algoritmo, se seleccionaría rrs_665 y se calcularía en memoria).Convertir los registros extraídos a formato GeoJSON.Respuesta: Objeto GeoJSON FeatureCollection donde cada feature tiene properties: {"turbidity_ntu": float}.GET /api/v1/turbidity/timeseriesDescripción: Retorna la serie de tiempo para un polígono o punto específico (para gráficos).Query Params:start_date, end_date (YYYY-MM-DD).polygon_wkt (string): Polígono en formato WKT para filtrar espacialmente (ej. usando ST_Intersects).algorithm (string): Enum [SVR].Respuesta: Array de objetos {"date": "...", "avg_turbidity_ntu": float}.
