Algoritmos Bio-ópticos (Lectura de Turbidez)Este documento especifica cómo el Agente Backend debe manejar la obtención de los valores de turbidez según el algoritmo seleccionado por el usuario en el frontend.Algoritmo 1: SVR (Support Vector Regression) - Fase 1En la Fase 1 del proyecto (MVP), el único algoritmo habilitado será el modelo propio basado en SVR.A diferencia de algoritmos empíricos clásicos, el cálculo de SVR no se realiza en el backend. El valor de turbidez resultante de este modelo ya viene precalculado e inyectado desde el archivo JSON proporcionado por el administrador.Lógica de Implementación (Backend):Cuando el usuario solicita datos usando el algoritmo SVR, el sistema actúa como un "passthrough" (puente).No se aplica ninguna fórmula matemática sobre el valor rrs_665.Se consulta directamente la columna tt_pred de la base de datos.El valor tt_pred se expone en la respuesta de la API bajo el alias estandarizado turbidity_ntu para que el frontend lo consuma sin alteraciones.Futuros Algoritmos (Fase 2)La arquitectura debe mantenerse flexible. En el futuro, se agregarán algoritmos que sí requerirán que el backend extraiga la columna rrs_665 y aplique operaciones algebraicas vectorizadas (usando NumPy/Pandas) para derivar la turbidez en tiempo real.Por ahora, el enum de algoritmos solo debe contener ['SVR'].
