Diseño de la Interfaz (Layout)Modo Pantalla Completa: El mapa (OpenLayers) ocupa el 100% de la pantalla (fondo).Panel Lateral (Izquierdo): Controles flotantes sobre el mapa.Selector de Algoritmo (Dropdown). Nota para Fase 1: Solo debe tener la opción "SVR (Support Vector Regression)" por defecto.Selector de Fecha (Date Picker). Oculta o desactiva las fechas que no tienen datos en la base de datos (Requiere endpoint que devuelva "días disponibles").Panel Inferior: Línea de tiempo (Timeline slider) para recorrer los meses/años y ver cómo cambia el Heatmap. Al reproducir (Play), pide los GeoJSON al backend iterativamente.Módulo Administrador (Ruta oculta /admin):Formulario de Drag & Drop para subir archivos JSON (con la nueva estructura que incluye TT_pred).Tabla de log de subidas.2. Lógica del Mapa (OpenLayers)El Agente Frontend NO debe usar Leaflet. Debe usar OpenLayers debido al volumen de puntos (700+) y la necesidad de interpolación.Renderizado del Heatmap:El frontend recibe el GeoJSON desde el Backend (vía GET /api/v1/turbidity/heatmap).Crea un VectorSource con este GeoJSON.Utiliza la clase Heatmap layer de OpenLayers.Peso (Weight): El peso de cada punto en el Heatmap debe estar vinculado directamente a la propiedad turbidity_ntu del GeoJSON (la cual el backend alimenta a partir del valor tt_pred de la base de datos). Debe normalizarse entre 0 y el valor máximo esperado de turbidez en la bahía.Gradiente de Color: Configurar un gradiente semántico. Ej: Azul (baja turbidez) -> Verde -> Amarillo -> Rojo/Marrón (alta turbidez/sedimentación).
